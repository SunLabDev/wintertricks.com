Quality Guidelines

All tricks must follow these quality guidelines

1. Write your trick as short as possible
This makes sure the content is easy to scan and the important bits stand out.

2. Include references to official documentation and resources
This makes sure the user can get further info on the details behind a trick.

3. Provide easy to understand example code
Try to provide real-world examples in your code examples where possible.
